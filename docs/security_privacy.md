Security & Privacy (Local-First)

- Local-only deployment by default; no outbound traffic unless explicitly enabled.
- Access control
  - Bind to localhost; optional user authentication for multi-user setups.
- Data protection
  - File permissions, checksums for memory objects, and tamper-evident logs.
- Backups & recovery
  - Immutable append-only logs; scheduled backups to local storage media.
- Privacy considerations
  - All memory content remains on-device; no cloud-sync by default.
- Compliance notes
  - Documentation to support local data governance and auditability.
